Ветка "Тест ветка" успешно добавлена в дерево (#390, дочерний узел "Ұлы жүз"). Вот все найденные проблемы в процессе:

1. Mind Map исчезает сразу после создания ветки После нажатия "Create" в модальном окне — дерево (Mind Map) полностью пропадает и показывается пустой контейнер. Дерево восстанавливается только после ручной перезагрузки страницы. Это значит, что после создания дерево не перерисовывается — нужно обновить данные и re-render. 2. Страницы /admin и /moderation не работают — редирект на главную При переходе по ссылкам "Admin" и "Moderation" из навигации происходит редирект на главную страницу /. Это означает что страницы либо не реализованы, либо сломана роутинг/авторизация для них. В навигации они отображаются, но открыть их нельзя.

3. Два разных варианта формы "Add Person" с несоответствиями Есть два места для добавления узла:
•	Кнопка + на узле в дереве (модальное окно) — Parent выбирается через выпадающий список, Access по умолчанию = Public
•	Страница /person/new (Add Person в навигации) — Parent через поиск, Access по умолчанию = Private Разные дефолтные значения Access и разный UX для одного и того же действия — путаница для пользователя. 4. После добавления ветки показывается страница-профиль вместо дерева После создания узла через модальное окно отображается карточка "Тест ветка" с кнопками Edit/View Tree прямо в блоке дерева — это нелогично. Ожидаемое поведение: модальное окно закрывается, дерево обновляется и показывает новый узел подсвеченным.

5. В модальной форме поле "Name" не помечено как обязательное (*) На странице /person/new поле Name отмечено звёздочкой *, а в модальном окне + — нет, хотя оно обязательное. 6. Узел дерева не показывает Designation, если оно пустое В карточке узла дерева отображается только имя и ID — без подсказки о пустых полях (Designation/History). У других узлов есть описание — у новых пусто и это не выделено визуально.
В ходе тестирования сайта familytree-frontend.fly.dev было выявлено несколько проблем разной степени критичности. Самая серьёзная проблема связана с добавлением ветки: после нажатия кнопки "Create" в модальном окне Mind Map полностью исчезает и показывает пустой контейнер — дерево не перерисовывается автоматически, и пользователю приходится вручную перезагружать страницу. Помимо этого, сразу после создания узла на странице появляется карточка-профиль новой ветки с кнопками Edit и View Tree прямо в блоке дерева, что выглядит неожиданно и нелогично. Страницы Admin и Moderation фактически не работают: при переходе по ссылкам из навигации происходит автоматический редирект на главную страницу, хотя пункты меню отображаются даже у авторизованного администратора. В системе существует два разных способа добавить узел — модальное окно по кнопке "+" на узле дерева и отдельная страница /person/new через навигацию — и они ведут себя по-разному. В модальном окне родитель выбирается через выпадающий список, а доступ по умолчанию выставлен как Public. На странице /person/new родитель ищется через поисковое поле, а доступ по умолчанию — Private. Такое расхождение создаёт путаницу. При вводе неверных данных на странице входа ошибка выводится на английском языке ("invalid credentials") даже если выбран казахский или русский интерфейс. При переключении языка весь интерфейс корректно переводится, однако содержимое узлов дерева — названия и описания родов — остаётся на казахском независимо от выбранного языка, поскольку данные хранятся в базе только на одном языке. Кнопка "Найти путь" не реагирует при пустых полях "От кого" и "К кому" — не отправляется запрос и не появляется никакого сообщения об ошибке. В форме регистрации поля Имя и Фамилия не помечены как обязательные, в отличие от Email и Пароля. В модальной форме добавления узла поле Name также не имеет маркера обязательности, хотя на странице /person/new оно отмечено звёздочкой. Навигация не адаптируется под мобильные экраны — все пункты меню остаются в одну строку без бургер-меню.
